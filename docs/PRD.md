## 2. 产品概述

### 2.1 产品定位

RS 是一款**智能个人知识助手**，类似 OPPO 小布记忆。它深度融入系统分享机制，用户可在任何 App 中将文字、图片、视频等内容一键分享至 RS，RS 自动调用多模态大模型解析内容并执行相应动作（记账、添加待办、创建日程、生成视频摘要等），让碎片信息被高效整理和记忆。

### 2.2 核心价值

- **一步完成“记录-分析-执行”**：系统级分享直达，无需在多个应用间跳转。

- **多模态智能理解**：能看懂图片账单、取件码、会议通知文字，甚至理解视频内容。

- **开放的大模型接入**：默认提供官方后端（内置模型），也允许高级用户配置自有 API Key 直接调用，保护数据隐私并控制成本。

- **全场景覆盖**：从生活琐事（记账、取件）到工作学习（日程、视频笔记）皆可管。

### 2.3 典型用户故事

1. **小明**在客服聊天界面截图一张带取件码的短信，点击分享→RS，RS 自动识别出“取件码 8-4-5021”，并在待办列表中创建“取快递：8-4-5021”，设置当天提醒。

2. **小红**收到一张火锅店消费小票，分享至 RS，RS 解析出“支出 ¥238，餐饮，2026-04-28”，一键确认后加入她的消费账本。

3. **大学生小刚**在班级群看到消息“明天上午 8 点运动场开运动会”，复制文字分享给 RS，RS 自动创建日历事件：“运动会”，4 月 30 日 8:00-12:00，并加入待办。

4. **阿强**在 B 站看到一个科普视频，分享链接到 RS，RS 后台解析视频内容生成 200 字中文摘要，方便稍后回顾。

## 3. 功能需求

### 3.1 系统分享入口接收

- **平台机制**

  - Android：注册 `ACTION_SEND` intent，支持 `text/plain`、`image/*`、`video/*` 等 MIME 类型。

  - iOS：实现 Share Extension，在系统分享菜单中显示 RS 的图标。

- **接收内容类型**

  - 单张/多张图片（JPEG/PNG/HEIC 等）

  - 纯文本（包括 URL、短信文字等）

  - 单个视频文件（mp4 等，文件大小后期限制，如 200MB）

  - 多个混合内容时，默认选取首个或列出供用户选择（V2）。

- **分享体验**

  - 用户选择 RS 之后，可以**直接后台处理**（不强制打开主 App），并弹出本地通知（或者流体云）告知“RS 正在分析…”。

  - 或立即唤起 RS 的小窗界面，进入“待处理”队列，以便用户实时查看结果。

  - 两种模式可在设置中选择：**静默处理（通知模式）** 或 **打开应用确认模式**。

### 3.2 智能内容分析引擎

- **统一意图识别 & 信息抽取**

  ```json
  {
    "action": "add_expense | add_todo | add_event | summarize_video | unknown",
    "confidence": 0.95,
    "data": { ... }   // 根据 action 变化
  }
  ```

- **各动作数据定义**

  - **账单**（add_expense）：

    - amount：金额（数字）

    - currency：币种（默认 CNY）

    - category：分类（如餐饮、交通、购物等，可自动推断）

    - date：消费日期（如无则用当前日期）

    - note：备注（商品详情等）

  - **待办**（add_todo）：

    - title：待办标题（如“取快递 8-4-5021”）

    - due_date：截止日期（可选）

    - reminder：是否设置提醒（默认提前 1 小时）

  - **日历事件**（add_event）：

    - title：事件名称

    - start_time：开始时间（ISO 8601）

    - end_time：结束时间（若无，默认开始+1 小时）

    - location：地点

    - notes：备注

  - **视频摘要**（summarize_video）：

    - title：视频标题

    - summary：摘要文本（200 字左右）

    - key_points：关键点列表（可选）

- **模型调用策略**

  - **图片/视频**：优先调用多模态模型（如 GPT-4V、Gemini Pro Vision）；若为视频链接，不直接给模型原始链接，而是由后端解析后再分析（见 3.4）。

  - **纯文本**：使用文本大模型（如 GPT-4o-mini），并解析自然语言中的日期、时间、金额、地点等实体。

- **用户自定义模型支持**

  - 在设置中可切换模型来源：

    - **使用官方后端**（默认）：所有请求通过 Go 后端转发，后端负责调用内置的模型 API（如 OpenAI / Azure / 国内合规模型）。

    - **自定义 API**：用户填写 API Base URL、API Key、模型名称。前端直接向该 URL 发起请求，数据不经过 RS 官方服务器，保护隐私。

  - 视频链接解析如需后端辅助（如 B 站 API 抓取），即使使用自定义模型，该“解析链接→获得视频信息/字幕”步骤仍会调用官方后端（用户可选择“仅本地处理视频文件”以完全绕过）。

### 3.3 动作执行结果处理

- **通用交互**

  - 原始分享内容缩略（图片缩略图/文字前 50 字）。

  - 识别动作及置信度。

  - 提取到的结构化信息（可编辑）。

  - 两个按钮：**确认执行** 和 **忽略/删除**。

- **具体执行**

  - **账单**：存入本地账本数据库，可查看列表、按月统计。

  - **待办**：存入待办列表，支持标记完成、删除。

  - **日历事件**：调用系统日历 API（Android: CalendarProvider, iOS: EventKit）添加事件，并在应用内同步显示。

  - **视频摘要**：保存摘要卡片，展示标题、摘要、来源链接，可归档、搜索。

- **未识别处理**

### 3.4 视频链接解析子模块（V2 核心功能）

- **目标**：支持主流视频平台（B站、YouTube、抖音等）的分享链接，提取视频基本信息及字幕/文字稿，交由大模型总结。

- **流程**

  1. 前端收到链接（`text/plain`），判断是否为匹配的视频平台 URL。

  2. 如果使用默认后端：

  - 将链接发送到 Go 后端服务。

  - 后端通过第三方解析器（如 yt-dlp、爬虫等）获取视频元数据、自动语音识别(ASR)字幕或直接下载字幕文件。

  - 将标题 + 文本内容拼接成 prompt，调用大模型生成总结。

  - 返回摘要 JSON 给移动端。

  3. 如果使用自定义 API：

  - 用户可选择**绕过后端**：自己下载视频文件并上传给大模型（受限于文件大小及模型能力）。

  - 或仍然使用后端解析（需将链接发给官方后端，但文本内容分析可使用自定义模型）。设置中提供开关。

- **限制**：视频解析耗时较长，后端采用异步任务，移动端通过轮询或推送获取结果。V1 可降级为仅支持已带摘要的链接（如某些平台分享时即有描述），或暂不支持，直接提示“视频摘要功能即将上线”。

### 3.5 数据管理

- **记忆库**：主界面按时间倒序展示所有处理过的“记忆卡片”，支持搜索、按类型筛选（全部、账单、待办、日程、摘要）。

- **编辑和删除**：长按卡片可编辑结构化字段、原始内容不可更改但可重新分析。

- **备份与同步**（V2）：本地 SQLite 存储，未来可选云同步至自建服务器或 WebDAV。

- **隐私清理**：一键清空所有数据，并确保后端已处理内容缓存同步清除。

### 3.6 设置与个性化

- **模型配置**

  - 默认/自定义模式切换

  - 自定义 API 设置：Base URL, API Key, Model Name

  - 连接测试按钮（发送简单 prompt 验证可用性）

- **分享处理方式**：静默通知 / 打开应用确认

- **默认提醒时间**：待办默认提醒（如提前 30 分钟）

- **分类标签管理**：自定义账单/待办的分类列表

- **关于与许可**：开源许可、隐私政策、使用条款。

## 4. 非功能需求

### 4.1 性能

- 分享接收响应 < 1 秒内拉起后台任务或通知。

- 文本分析（调用模型）应在 5 秒内返回结果（视网络和模型速度）。

- 图片/视频分析可在 15 秒内给出粗结果（通过后端队列优化）。

- 应用冷启动 < 2 秒，页面滑动 60fps。

### 4.2 安全与隐私

- 用户自定义 API Key 必须加密存储在设备安全区域（Android Keystore / iOS Keychain），通过 `flutter_secure_storage` 实现。

- 官方后端传输全部使用 HTTPS；用户上传的图片/视频文件在分析完成后立即从服务器磁盘删除，仅在内存中处理。

- 后端不记录任何明文内容，只保留匿名统计（如分析数量、成功类型分布），且不关联用户身份。

- 应用需提供清晰的隐私声明，说明数据在何时被上传。

- 官方后端模型 API 调用需用户认证（JWT）以进行配额管理和防滥用。用户注册支持邮箱/手机号，或匿名使用受限额度。

### 4.3 兼容性

- Android 12及以上，适配 Material You 动态色彩。

- iOS 15.0 及以上，适配暗黑模式。

- 适配手机和平板布局。

### 4.4 可扩展性

- 后端 Go 服务设计为微内核+插件式 AI 模型适配器，可快速接入新模型（Claude、文心一言等）。

- 新动作类型（如“添加联系人”、“设置提醒”）通过模型输出 `action` 类型及对应 `data` schema 扩展，客户端只需增加对应的执行器和 UI 卡片。

## 5. 系统架构与数据流

### 5.1 高层架构图

```
[分享源 App] 
    | 系统分享菜单
[Q App 分享入口] (Android: ShareActivity / iOS: ShareExtension)
    | 通过 Method Channel
[Flutter 主引擎] 
    ├── 配置中心（默认后端/自定义API）
    ├── 内容处理器（调用 AI 服务）
    │   ├── 方式一： -> [Go 后端] -> [默认大模型 API]
    │   └── 方式二： -> [用户自定义大模型 API] 
    └── 动作执行器 & 本地数据库
         ├── 账单模块
         ├── 待办模块
         ├── 日历模块
         └── 摘要模块
```

### 5.2 分享数据接收流程详解

1. 用户在任意 App 中点击“分享”，选择 RS。

2. 系统创建对应的 Intent/Extension，并唤醒 RS 的特定原生界面（可透明）。

3. 原生代码提取分享的内容 URI / 文本，通过 Method Channel 传给 Flutter 层。

4. Flutter 层显示一个轻量处理界面（或直接根据设置进行后台操作）。

5. **模型调用**：

- 若为**默认后端模式**，将文件上传到 Go 后端（临时存储）并发送分析请求，后端调用大模型返回结构化 JSON。

- 若为**自定义模式**，则前端直接发送 Base64 图片或多部分表单到用户指定的 API，附带系统 prompt。

6. 解析返回 JSON，在本地生成记忆卡片预填充项，等待用户确认（或自动执行，取决于设置）。

### 5.3 后端 API 设计（Go）

**基础路由**（V1）：

- `POST /api/v1/analyze`：接收多部分表单（文本、图片文件、视频链接等），返回 JSON 分析结果。

  - `text`：纯文本内容（可选）

  - `images`：图片文件（可选，最多 3 张）

  - `video_url`：视频链接（可选）

  - `model`：可选，指定要使用的模型（管理员预先配置）

  ```json
  {
    "action": "add_expense",
    "confidence": 0.96,
    "data": { ... }
  }
  ```

- `POST /api/v1/video/summarize`：接收视频链接，返回摘要（异步，先返回任务 ID，然后轮询 `GET /api/v1/tasks/{id}`）。

- `POST /api/v1/auth/register`, `POST /api/v1/auth/login` 等用户认证接口。

- 同时提供 `/api/v1/usage/me` 查询当前配额。

**内部模块**：

- 模型适配器层：根据配置路由到 OpenAI、Anthropic、百度文心等。

- 视频解析模块：封装 youtube-dl 或平台专用爬虫。

- 文件处理器：图片压缩、格式转换等。

- 任务队列（Redis）：处理长耗时视频摘要任务。

### 5.4 数据模型（本地 SQLite）

核心表 `memories`：

| 字段                  | 类型          | 说明                                        |
| ------------------- | ----------- | ----------------------------------------- |
| id                  | INTEGER PK  | 自增ID                                      |
| type                | TEXT        | 'bill','todo','event','summary','unknown' |
| raw_content_type    | TEXT        | 'text','image','video'                    |
| raw_content_summary | TEXT        | 缩略文本或本地缩略图路径                              |
| structured_data     | TEXT (JSON) | 解析后的结构化信息                                 |
| created_at          | DATETIME    | 创建时间                                      |
| status              | TEXT        | 'pending','confirmed','dismissed'         |
| source_app          | TEXT        | 可选，来源 App 名称                              |

`type` 为 `bill` 时，扩展存储表 `expenses`；`todo` 对应 `todos`，以此类推，或所有字段均用 JSON 存储，以支持灵活扩展。

## 7. 技术实现注意点

### 7.1 Flutter 分享入口实现

- **Android**：在 `AndroidManifest.xml` 中为特定 Activity 添加 intent-filter：

  ```xml
  <activity android:name=".ShareActivity">
      <intent-filter>
          <action android:name="android.intent.action.SEND" />
          <category android:name="android.intent.category.DEFAULT" />
          <data android:mimeType="text/plain" />
          <data android:mimeType="image/*" />
          <data android:mimeType="video/*" />
      </intent-filter>
  </activity>
  ```

  该 Activity 内嵌一个 FlutterFragment 或直接用 Method Channel 传递数据给 Flutter。

- **iOS**：创建 Share Extension target，在 `ShareViewController` 中通过 App Group 共享容器或直接启动主 App URL Scheme 传递数据（需处理苹果审核要求）。

- 推荐使用成熟插件如 `receive_sharing_intent`，减少原生代码维护成本。

### 7.2 大模型调用与安全

- 前端对用户自定义 API 直接发送 HTTPS 请求，注意使用 `dio` 等库设置超时、重试，图片需先压缩（限制 2MB 内）。

- 后端默认模型网关需做好速率限制、输入内容过滤（防止滥用）。

- API keys 不硬编码在客户端，官方后端密钥仅在服务器环境变量中。

### 7.3 隐私合规

- 应用内提供“数据不上传官方服务器”的自定义模式说明。

- 即使使用官方后端，也显示“内容将加密传输并在一小时内删除”提示。

- 遵守 Apple 和 Google 关于用户生成内容、AI 生成内容的政策。

## 9. 风险与应对

- **模型幻觉导致错误执行**：所有自动创建的内容都需要用户二次确认（默认开启，可关闭）；提供置信度阈值设定。

- **视频解析版权与合规**：后端仅解析可公开访问的网页元数据，不破解加密内容，遵守 robots.txt。用户上传视频分析需获权。

- **成本控制**：默认后端设置每日每用户最大调用次数（如 10 次），超额需升级订阅；前端直调由用户自行承担费用。

- **iOS 审核对分享扩展限制**：确保 Extension 不执行过重的操作，将主要逻辑交给主 App 处理，Extension 仅收集数据并唤醒主 App。

## 10. 附录：术语表

- **多模态模型**：能够同时理解文字、图片、视频等非文本信息的大语言模型。

- **Share Extension**：iOS 系统的扩展机制，允许应用在系统分享菜单中出现并接收数据。

- **Intent (Android)**：描述应用操作的一种消息机制，`ACTION_SEND` 用于接收分享内容。

- **置信度**：模型对判断结果确定性的评分，0-1 之间。
